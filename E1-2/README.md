# 퀴즈 게임 (Quiz Game)

## 1. 프로젝트 개요
이 프로젝트는 Python 기초 문법과 객체 지향 프로그래밍(OOP)을 활용하여 제작한 콘솔 기반의 퀴즈 게임 프로그램입니다. 사용자가 퀴즈를 풀고 점수를 기록하며, 새로운 문제를 직접 추가하여 나만의 퀴즈 데이터를 구축할 수 있는 기능을 제공합니다.

## 2. 퀴즈 주제 선정 이유
- **주제**: 과일
- **이유**: 남녀노소 누구나 쉽게 이해할 수 있는 친숙한 주제이며, 사물에 대한 명확한 특징(색상, 모양, 계절 등)이 있어 4지선다형 문제를 구성하기에 적합하여 선정하였습니다.

## 3. 실행 방법
프로그램을 실행하기 위해서는 **Python 3.10 이상** 버전이 필요합니다.
1. 터미널(또는 CMD)을 열고 프로젝트 폴더로 이동합니다.
2. 아래 명령어를 입력하여 프로그램을 실행합니다.
   ```bash
    python main.py
   ```

## 4. 기능 목록
- **퀴즈 풀기**: 저장된 퀴즈를 순차적으로 출제하며, 정답 여부를 즉시 확인하고 최종 결과를 표시합니다.
- **퀴즈 추가**: 사용자가 직접 문제, 4개의 선택지, 정답 번호를 입력하여 새로운 퀴즈를 등록합니다.
- **퀴즈 목록**: 현재 시스템에 등록된 전체 퀴즈의 질문 목록을 확인합니다.
- **점수 확인**: 역대 최고 점수(가장 많이 맞힌 정답 수)를 확인합니다.
- **종료 및 저장**: 프로그램을 종료할 때 추가된 퀴즈와 최고 점수를 `state.json` 파일에 안전하게 저장합니다.
- **예외 처리**: 숫자 변환 실패, 범위 밖 숫자, 빈 입력 등에 대한 예외 상황을 처리하여 비정상 종료를 방지합니다.

## 5. 파일 구조
- `main.py`: 프로그램의 메인 로직, `Quiz` 클래스 및 `QuizGame` 클래스가 포함된 소스 코드 파일
- `state.json`: 퀴즈 데이터와 최고 점수가 저장되는 JSON 형식의 데이터 파일
- `README.md`: 프로젝트 설명 및 사용 가이드 문서

## 6. 데이터 파일 설명 (state.json)
프로젝트 루트의 `state.json` 파일은 UTF-8 인코딩으로 데이터를 관리하며, 아래와 같은 스키마 구조를 가집니다.

| 키(Key) | 타입(Type) | 설명 |
| :--- | :--- | :--- |
| **`quizzes`** | Array (List) | 등록된 퀴즈 객체들의 목록. 각 객체는 `question`, `choices`(배열), `answer`(정답 번호)를 포함함 |
| **`best_score`** | Integer | 사용자가 기록한 역대 최고 정답 개수 |



## 7. 학습 경험 및 핵심 개념 정리

### 1) Python 기초
* **변수**: 데이터를 저장하는 이름 붙은 공간이며, 코드의 재사용성과 가독성을 높이기 위해 사용합니다.
* **데이터 타입**: 정수(`int`), 문자열(`str`), 논리형(`bool`), 순서가 있는 목록(`list`), 키-값 쌍의 사전형(`dict`)으로 구분하여 데이터를 효율적으로 처리합니다.
* **조건문**: `if/elif/else`를 사용하여 특정 조건 충족 여부에 따라 프로그램의 실행 흐름을 제어합니다.
* **반복문**: 정해진 횟수나 범위는 `for`를 사용하고, 특정 조건이 유지되는 동안은 `while`을 사용하여 반복 작업을 수행합니다.
* **함수**: 반복되는 로직을 독립된 블록으로 정의하며, 매개변수(입력)와 반환값(출력)을 통해 코드의 모듈화를 실현합니다.

### 2) 클래스와 객체 (OOP)
* **클래스**: 데이터(속성)와 행동(메서드)을 하나로 묶은 설계도이며, 복잡한 시스템을 구조적으로 설계하기 위해 사용합니다.
* **`__init__` & `self`**: `__init__`은 객체 생성 시 초기화를 담당하는 메서드이며, `self`는 생성된 객체 자기 자신을 가리키는 변수입니다.
* **속성 & 메서드**: 객체가 가지는 특징은 속성(Attribute)으로, 객체가 수행하는 기능은 메서드(Method)로 정의하여 활용합니다.

### 3) 파일 입출력 및 예외 처리
* **파일 입출력**: 파일을 열고(`open`), 데이터를 읽거나 쓴(`read/write`) 뒤 닫는 과정을 통해 데이터를 영구적으로 저장합니다.
* **JSON**: 텍스트 기반의 경량 데이터 형식으로, 구조화된 데이터를 파일에 저장하거나 네트워크로 주고받기에 용이하여 사용합니다.
* **`try/except`**: 프로그램 실행 중 발생할 수 있는 오류를 미리 예상하고 처리하여 시스템의 비정상 종료를 방지합니다.

### 4) Git 기초 및 협업
* **Git**: 소스코드의 변경 이력을 관리하는 버전 관리 시스템으로, 협업과 코드 복구 및 관리를 위해 필수적입니다.
* **주요 명령어**: `init`(저장소 시작), `add`(변경사항 스테이징), `commit`(버전 기록), `push`(원격 업로드), `pull`(원격 변경사항 가져오기), `checkout`(브랜치 전환), `clone`(저장소 복제) 등을 사용합니다.
* **브랜치 & 병합**: 독립적인 작업 공간을 생성(`branch`)하여 작업한 뒤, 완성된 결과물을 메인 코드에 합치는(`merge`) 프로세스를 수행합니다.
