# 🧠 Mini NPU Simulator 개발 리포트

## 1. 프로젝트 개요
컴퓨터가 시각적 데이터를 숫자 배열로 인식하고 처리하는 핵심 원리인 **MAC(Multiply-Accumulate) 연산**을 시뮬레이션하는 프로그램입니다.3×3부터 25×25까지 다양한 크기의 필터와 패턴을 비교하여 유사도를 판별하고, 데이터 크기 증가에 따른 연산 성능 변화를 분석합니다.

## 2. 실행 방법
프로그램은 **Python 3.8 이상** 환경에서 외부 라이브러리 설치 없이 실행 가능합니다.
1. 터미널에서 프로젝트 폴더로 이동합니다.
2. 아래 명령어를 입력하여 프로그램을 실행합니다.
   ```bash
   python main.py
   ```
3. 메뉴 선택창이 나타나면 숫자를 입력합니다.
   - **1**: 사용자 직접 입력 모드 (3×3 고정)
   - **2**: `data.json` 분석 모드 (다양한 크기 일괄 처리)

## 3. 구현 요약
### 3.1. MAC(Multiply-Accumulate) 연산
- 외부 라이브러리(NumPy 등)를 전혀 사용하지 않고, **순수 파이썬 이중 반복문**을 사용하여 위치별 곱셈과 전체 누적 합을 직접 계산하도록 구현하였습니다. [cite: 454-456]
- $O(N^2)$ 시간 복잡도를 가지며, 각 크기별로 **10회 반복 측정**하여 평균 연산 시간을 산출함으로써 측정의 신뢰도를 높였습니다.

### 3.2. 라벨 정규화(Normalization) 정책 
- 다양한 데이터 소스에서 오는 라벨 형식(`+`, `x`, `cross`, `Cross` 등)을 프로그램 내부 표준인 **`Cross`**와 **`X`**로 강제 변환하여 데이터 일관성을 확보하였습니다.

### 3.3. 동점 처리 및 허용 오차(Epsilon) 정책
- 부동소수점 연산 시 발생하는 미세한 오차로 인한 잘못된 판정을 방지하기 위해 **`1e-9`** 기준의 허용 오차 정책을 적용하였습니다.
- `abs(Score_A - Score_B) < 1e-9`인 경우 어느 한쪽의 우세를 판정하지 않고 **`UNDECIDED`**로 처리합니다.

## 4. 결과 리포트 (분석 및 고찰)

### 4.1. 실패 원인 분석 (FAIL Case Analysis)
본 시뮬레이터에서 발생할 수 있는 주요 실패(FAIL) 원인은 다음과 같이 설계 및 대응되었습니다.
- **데이터 불일치**: `data.json`의 필터 크기와 패턴 크기가 다를 경우, 프로그램 중단 없이 해당 케이스만 `FAIL`로 처리하고 사유를 기록합니다.
- **라벨 오기입**: 정규화되지 않은 라벨(`+`, `x`)은 내부 함수가 자동으로 표준화하므로 라벨 불일치로 인한 `FAIL` 확률을 최소화했습니다. 
- **동점 판정 실패**: 기대값(`expected`)은 `X`인데 계산 결과가 `UNDECIDED`가 나온 경우(예: `1e-9` 이내의 점수 차이), 수치적 엄격함에 의해 `FAIL`로 분류됩니다. 이는 데이터의 모호성을 반영하는 로직의 결과입니다. 

### 4.2. 시간 복잡도 분석 및 성능 해석
- **$O(N^2)$ 근거**: MAC 연산은 $N \times N$ 크기의 행렬에서 각 요소를 한 번씩 방문하므로 총 $N^2$번의 곱셈 연산이 발생합니다.
- **데이터 기반 해석**: 
  - **3×3** 연산 횟수: $3^2 = 9$회
  - **25×25** 연산 횟수: $25^2 = 625$회
- 측정 결과, 데이터 크기가 약 8배($3 \to 25$) 증가할 때 연산 횟수는 약 69배 증가하였으며, 실제 측정된 평균 연산 시간도 이와 비례하여 기하급수적으로 늘어나는 양상을 보였습니다. 이는 하드웨어의 병렬 처리 없이 직렬 반복문으로 수행할 때 발생하는 전형적인 $O(N^2)$의 특징입니다.

## 5. 데이터 파일 구조 (`data.json`)
- **filters**: `size_5`, `size_13`, `size_25` 등 규격별 필터(Cross, X) 정의.
- **patterns**: 각 패턴의 입력 배열(`input`)과 기대 결과(`expected`)를 포함.
- **스키마 검증**: 패턴 키의 이름에서 크기를 추출하여 해당 규격의 필터와 자동으로 매칭하도록 구현하였습니다.
